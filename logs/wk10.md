# Week 10 Log

## What we covered
- A live build of a Chrome extension with Claude Code — empty folder to
  loaded-and-working in about twenty minutes. The point wasn't the extension:
  an extension is just a folder of HTML, CSS, and JS plus a `manifest.json`
  telling Chrome what it is.
- The real lesson was what happens *after* the code runs: read the spec the
  agent writes before you read its code, be suspicious of tests the agent
  wrote for itself (all green while it was still counting HTML comments as
  words), and ask it to list every decision it made that you didn't specify.
- `CLAUDE.md` as a standing-instructions file the agent reads every session,
  so you write down the behaviour you want instead of hoping for it.
- Commit before you let an agent run. `git add -A && git commit` first, so
  `git checkout .` undoes a bad run in one command and `git diff` shows you
  exactly what changed.

## What I built
- Built my final project, **football-impostor** — a pass-and-play party game
  where a group shares one device, everyone sees the same footballer, and one
  (or two, or three) players are secretly told they're the impostor.
- Structure: `js/data.js` exports the footballer list, `js/main.js` holds all
  game logic, loaded with `<script type="module">` — the data/logic split from
  the code-organization exercise.
- Features built this week: setup screen with editable player names (defaulting
  to Player 1, 2, 3...), a choice of 1–3 impostors, the pass-and-play reveal
  where each player taps to see their role privately and hides it before
  passing, a randomly chosen player to start the discussion, and a reveal
  button that shows who the impostors were and which footballer it was.
- Added a "how to play" rules box on the setup screen so a new group can start
  without being talked through it.
- Deployed to GitHub Pages, then added a second game on its own page — Career
  Paths, where a hidden footballer's clubs are revealed one at a time and you
  guess who it is, with a name search, five guesses, and a photo that un-blurs
  on the reveal.

## What I learned
- Fetch data at build time, not runtime. A one-off Node script calls the
  football API on my machine and writes a static data file, so the deployed
  site makes no API calls — no key in public code, nothing to fail mid-game.
  Same reason I scoped out live multiplayer: a static site has nowhere to keep
  shared state, and passing one phone enforces secrecy for free.
