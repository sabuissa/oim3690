# Week 9 Log

## What we covered
**Monday 7/13 — Browser APIs + real-world tech stack**
- Browser APIs: free capabilities the browser gives you without installing
  anything — localStorage (remembers small things across refreshes),
  setTimeout/setInterval (do something later, or repeatedly), geolocation,
  clipboard, text-to-speech, and file upload.
- localStorage stores text only, so objects/arrays need JSON.stringify. Never
  put passwords or API keys in it, and don't treat it as the source of truth —
  it lives in one browser on one machine.
- When a project outgrows GitHub Pages (hiding a key, storing shared data),
  you add one piece rather than starting over: Vercel or Cloudflare for
  server-side code, Supabase for a real database.
- Scope is what you leave out on purpose. Naming what the project deliberately
  does NOT do is the defense against a project that grows until it can't be
  finished.

**Wednesday 7/15 — MP3 + final project idea showcase, proposal peer review**
- Everyone demoed MP3 live, named the API behind it, and pitched a final
  project idea in one sentence, then gave each other feedback in breakout
  groups (one feature to cut, one thing missing).
- Two people had an API die mid-project — one deprecated and started demanding
  a key, one ran out of free credits. Lesson: test an API on its own before
  building around it, and show a clear message when a call fails so a dead API
  still tells the visitor something.
- "Free with no key" is a real requirement with one week left.
- A specific real user is the cheapest source of originality.
