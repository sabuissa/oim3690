My Learning Journey

OIM3690 — Sultan Abuissa

Ten weeks ago I didn't know what a GitHub branch was. Now I've shipped a two-game final project where an AI agent does most of the typing and I decide what it's allowed to build. This page is the path between those two points, pulled from my ten weekly logs in logs/.

Week by Week

Week 1Foundations

Set up GitHub and published my first live site on GitHub Pages. Didn't understand CSS, branches, or how to write HTML without help yet — and said so.

Week 2Foundations

Got the actual oim3690 repo organized — logs folder, all the required files. Committing and pushing with GitHub Desktop finally clicked.

Week 3Styling & Forms

Moved CSS into its own file, learned element/ID/class selectors, and built contact.html with a real POST/GET form. Started MP1.

Week 4Styling & Forms

Reused one stylesheet across every page instead of rewriting styles each time. Still no idea how to make any of it respond to mobile.

Week 5JavaScript Starts

Studied the in-class JS demo line by line — addEventListener, a parseInt fallback, template literals — and wrote the explanations myself after Claude walked me through it.

Week 6JavaScript Starts

Built my first real interactive app: a to-do list, with the JS pulled into its own file. First time DOM methods felt like something I was using on purpose, not copying.

Week 7Arrays & Loops

Arrays, for/for...of/forEach, and the pattern under basically every real app: loop over an array, add an if. Watched an infinite loop freeze the browser. Started MP3.

Week 8APIs & Security

Fetched real data for the first time — a joke API, a class API with a token, the OpenAI API for a mini chat app. Learned .gitignore the hard way: config.js holds a key, so it never gets committed. Also the week I stopped taking AI output at face value — caught it trying to commit stray Python I never wrote.

Week 9APIs & Security

Learned what a browser can already do without installing anything (localStorage, geolocation) and the discipline of scope — deciding what a project deliberately won't do. Scoped my final project, football-impostor, on purpose: no live multiplayer, no backend, hardcoded data. Split data from logic with export/import for the first time.

Week 10Final Project

Watched a Chrome extension get built live in twenty minutes and learned the real lesson wasn't the extension — it was reading the agent's spec before its code, distrusting tests it wrote for itself, and committing before letting it run so a bad session is one git checkout away from gone. Shipped football-impostor: pass-and-play role reveal, editable player names, and a second game, Career Paths. Fetched all the football data at build time, not runtime, so the live site holds no API key and has nothing that can fail mid-game.

Week 1 vs. Week 10

Week 1

Didn't know what a branch was. Needed AI to explain what each file in a repo was even for. My "app" was a static HTML page I could barely style.

→

Week 10

Direct a Claude Code agent through a two-game project: hand it a spec and a data/logic boundary, let it write the code, then read its output like I'd check a teammate's — and know enough to catch it when its own tests are lying to me.

The AI didn't change over the semester — I did. In Week 1 I was asking it to explain what HTML tags were. In Week 10 I gave it a rule ("fetch at build time, not runtime") and a boundary ("data here, logic there") and let it write the code, because I'd finally learned enough to tell whether what it built was actually right. That's the real skill this class taught me — not JavaScript on its own, but supervising the thing that writes it.
